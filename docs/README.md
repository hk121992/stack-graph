# Design docs

This directory holds stack-graph's **design history and decision records** — the *why*
and the *how it got here*. It complements, and is deliberately kept separate from, the
[handbook](../handbook/content/), which holds the **canonical specifications** — *what is
true now*, kept terse.

- **[design-history.md](design-history.md)** — the narrative: how the model was arrived
  at, theme by theme, including the turns and corrections along the way.
- **[decisions.md](decisions.md)** — the decision log: each locked decision with its
  rationale, the alternatives weighed, and its status.

Rule of thumb: if it states *what the system is*, it belongs in the handbook; if it
explains *why we chose it* or *what we rejected*, it belongs here.

These are **working documents** — append as the design evolves, and supersede rather than
delete (the rejected turn is part of the record).
