# Research-Backfill Reconciliation — gaps filtered against locked decisions

**Input:** the 84 HIGH + MEDIUM gaps from `docs/research-backfill-triage.md` (27 high, 57
medium; the 50 lows were not reconciled — deferred as polish).
**Method:** five parallel reconcilers (one per node-cluster), each reading the full
`docs/decisions.md` (D1–D56) + the relevant spec/design docs, then judging every gap
**APPLY / DROP / REJECT** against what is already locked. Evidence standard: DROP/REJECT
requires a concrete citation; absent a citation, the gap stays APPLY.

## Headline

| Verdict | Count | Meaning |
|---|---|---|
| **APPLY** | **75** | Genuine open gap — becomes a real amendment |
| **DROP** | **8** (+1 stale-premise) | Already decided/specified upstream; node conforms — do **not** amend (some reduce to a one-line cross-ref) |
| **REJECT** | **1** | Recommendation conflicts with a locked decision; applying it would **regress** the design — node stays as-is |

**The catalogue of genuine work shrank only modestly (84 → 75), but the inversion is the
story:** both gaps the original triage escalated to *"operator must decide"* (measure-outcomes
CF-1, product-dashboard-curator CF-5) turned out to be **non-gaps** — already settled by D49 +
the analytics/artefacts spec. The triage's external-corpus lens was accurate at finding
*mechanical thinning* but blind to *deliberate* generalisation; every DROP/REJECT below is a
place the node dropped something **on purpose** per a locked decision. We now know the 75
amendments won't regress the design.

## Decisions that EMERGED from reconciliation (operator-relevant)

These weren't in the original triage's "operator decision" list — they surfaced when the gaps
were cross-checked against the spec and each other. **These, not the triage's two escalations,
are the real forks to settle before the amend wave.**

1. **Author the `decisions-store` reference first (prerequisite).** Named by `measure-outcomes`,
   `log-decision`, `capture-learnings`, and `reconcile`, but it doesn't exist in `graph/_refs/`.
   Several APPLYs and the reconcile-CF-2 wiring depend on it. (Its sibling `instrumentation-preamble`
   already exists — only `decisions-store` is missing.) → **author it before the dependent edges.**

2. **Specify the deploy-event artefact ONCE, in the analytics event log.** `deploy C3` and
   `land CF-3` both ask for a per-deploy timing/report artefact (the DORA/MTTR contract — triage
   theme #2). Its home should be an **extension to the `06-analytics` local event log / stream
   namespacing**, not a bespoke `.gstack/deploy-reports/` store, and it should be defined once and
   consumed by both nodes. → **decide the event-shape extension before two nodes implement it
   separately.** (Note: this confirms theme #2 is *partly* genuine — measure-outcomes' half was
   already solved via debrief/recorder; the deploy/land half is real.)

3. **Severity-scale consistency for non-lens findings.** `severity-scale.md` (D33) mandates
   **P0–P3** but is scoped to *review lenses*. `spec-diff`, `reconcile`, and `drift-detector` emit
   **low/med/high** and are explicitly **not lenses**. → **decide:** does P0–P3 extend to non-lens
   findings, or do those nodes keep their own scale? (Affects spec-diff CF-1, reconcile CF-1,
   drift-detector CF-1.)

4. **Cross-node de-duplication — land these as single amendments, not duplicates.**
   - `reconcile CF-5` (CHANGED status) **=** `spec-diff CF-1`'s CHANGED half — one amendment to the
     spec-diff output contract, consumed by reconcile.
   - `land CF-2` (revert) + `land CF-4` (post-deploy smoke check) resolve against `deploy C5` + `deploy C4`.

5. **Two smaller open questions to resolve while amending:** `capture-learnings CF-4` (where the
   prior-proposals list lives — reconcile against `artefacts-design §1`: committed `workspace/<surface>/`
   vs gitignored `.stack-graph/`); `harness-init C2` (the harness ambient-instructions surface — pair
   with the node's own open design question #3).
