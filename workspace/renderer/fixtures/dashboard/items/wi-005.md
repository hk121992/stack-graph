## Problem

The factory has no modelled user journey layer. The experience-thread design (`docs/experience-thread-design.md`) lays out persona-JTBD-scenario modelling, but the corresponding graph nodes have not been authored and the integration with the product-dashboard outcome layer is not yet specified.

## Opportunity

Completing the experience thread unlocks the full PM pack: the `simulate-users` node can run meaningful scenarios against real personas, and the `product-lens` can draw on concrete JTBD evidence for risk_state assessments.

## Open questions

- Which personas does the stack-graph factory need to model at pre-launch maturity?
- What is the integration seam between the experience-thread and the `product-dashboard-curator`?
- Is a separate `experience-contract` format needed, or can the VPC schema cover it?
